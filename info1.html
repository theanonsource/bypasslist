Обход глобального межсетевого экрана цензуры

Клиенты

NekoRay - NekoRay Github releases

Подойдет для Linux и Windows, но есть отдельный софт и на Android. Инфрастурктура на ядре sing-box. Для стабильной работы обязательно включить TUN Mode. В настройках TUN Settings включить TUN Enable IPv6 (избавиться от утечки IPv6).

Hiddify - Hiddify Github releases

Кроссплатформенный. Инфрастурктура на ядре xray. Настройка для правильного функционирования: режим - VPN, Enable TLS fragment - включить, реализация TUN - mixed.

v2rayNG - v2rayNG Github releases

Поддерживает Android. Инфрастурктура на ядре v2ray/xray. Настройка для правильного функционирования: Настройки > Раздел Настройки фрагментирования > Использовать фрагментирование (tls-фрагментирование повышает сложность анализа трафика системой цензуры)

Руководство

1. Соберите клиент (установите и настройте его, см. Клиенты).
2. Скопируйте указанную ссылку на VLESS конфиги (см. Доступные виртуальные сервера) и вставьте её в клиент для импорта серверов.
Для v2rayNG: нажмите + > "Импорт из буфера обмена".
Для v2rayNG: выберите Server > Add profile from clipboard.
Для Hiddify: нажмите + > "Добавить из буфера обмена".
3. Отсортируйте полученные сервера по задержке с помощью функционала клиента.
Для v2rayNG: выберите "Время отклика профилей группы" (подождите несколько минут, это зависит от мощности устройства и количества серверов) > "Сортировка по результатам теста".
Для v2rayNG: выберите Server > Current Group > Url Test (ждите). Затем отсортируйте по вкладке Test Results.
Для Hiddify: сортировка происходит в реальном времени, что может нагрузить систему.
4. Проверьте функциональность сервера через веб-ресурсы: https://browserleaks.com/dns, https://2ip.io/privacy.

Серьезно и безотлагательно отнеситесь к проверке конфига, который хотите использовать. Обследуйте свою виртуальную сеть, проинспектируйте SSL traffic через ресурс https://www.doileak.com/classic.html. Если замечены проблемы с SSL рекомендуется сменить сервер. Поскольку VLESS протокол не шифруемый напрямую, трафик может течь через TLS. Таким образом, наличие TLS даёт показатель безопасности. В противном случае, если конфиг не имеет TLS показателя, стоит усомниться в защищенности соединения. Из этого следует: если есть необходимость в безопасном соединении, проверяйте конфиги серверов на наличие TLS, такие сервера более безопасные. Рекомендуется периодически обновлять лист серверов ("группу профилей" или "подписку группы" и т.п.), ибо сервера могут истекать сроком действия или быть неактуальными с течением времени. Если сервер активен - это еще не означает, что он функционирует в текущих условиях сети, где система цензуры постоянно анализирует трафик. Проверяйте IP через браузер. TLS - протокол, используемый поверх в протоколе VLESS для улучшения безопасности и конфиденциальности соединений. SSL-трафик в контексте VPN относится к использованию протокола SSL (Secure Sockets Layer) для шифрования данных, передаваемых через виртуальную частную сеть. Это обеспечивает безопасность и конфиденциальность данных, передаваемых между клиентом и сервером.
