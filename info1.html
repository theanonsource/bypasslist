Обход глобального межсетевого экрана цензуры

Результативный обход цензуры ТСПУ и DPI, обходит фильтры на базе черного списка и DPI.

Опубликовано / Последнее изменение: 04.10.25

В наше время безопасность поделилась на два лагеря: для пользователя и для контроля. К сожалению, гегемону очень неудобно, если пользователь будет организовывать собственную информационную безопасность, для него это рассматривается как потенциальная национальная угроза...
Для любого человека VPN с 2025 года становится важным инструментом безопасности и конфиденциальности. Без частной сети теперь невозможно представить собственную безопасность. В сети распространяются VPN-сервисы, которые очень быстро развиваются в росте использования, но ровно с таким же успехом любой желающий злоумышленник способен создать свой сервис и начать продвигать в суматохе. Пользуясь VPN-сервисами, особенно в наше время, когда этих сервисов прямо-таки обилие, вы подвергаете себя информационной опасностью. Обход блокировок, несомненно, обогащает пользовательский опыт в сети, где каждый клик - потенциальная опасность, но нужно помнить о отсутствии гарантий безопасности таких распространённых решений.
На этой странице размещена информация, как организовать виртуальную частную сеть с помощью несвязанных между собой ресурсов, как можно более децентрализовано. Именно на вас ложиться отвественность собственной безопасности, ниже описано, как проверить безопасность конфига и соединения.
Децентрализация — это единственное конфиденциальное спасение.

Клиенты

NekoRay - NekoRay Github releases • NekoRay Github Android releases

Подойдет для Linux и Windows и Android. Инфраструктура на ядре sing-box. Для стабильной работы обязательно включить TUN Mode. В настройках TUN Settings включить TUN Enable IPv6 (избавиться от утечки IPv6). Чтобы избавиться от DNS утечки нужно включить System Proxy. Функция Enable DNS Routing должна быть включена: Routing Setting > DNS > Enable DNS Routing.
Для Android: Настройки > Входящие настройки > Добавить HTTP-прокси к VPN (на Android 10+ могут возникать проблемы с соединением, данная функция эту проблему решает).

Hiddify - Hiddify Github releases • Hiddify Github IOS version

Кроссплатформенный. Инфраструктура на ядре xray. Настройка для правильного функционирования: режим - VPN, Enable TLS fragment - включить, реализация TUN - mixed.

v2rayNG - v2rayNG Github releases

Поддерживает Android. Инфраструктура на ядре v2ray/xray. Настройка для правильного функционирования: Настройки > Раздел Настройки фрагментирования > Использовать фрагментирование (фрагментирование пакетов повышает сложность анализа трафика системой цензуры)

Доступные виртуальные сервера

Source VLESS-configs - более тысячи VLESS-конфигов от barry-far

Конфиги собираются автоматически с отобранных различных источников, список конфигов обновляется каждые ~15 минут.

Руководство

1. Соберите клиент (установите и настройте его, см. Клиенты).
2. Скопируйте указанную ссылку на VLESS конфиги (см. Доступные виртуальные сервера) и вставьте её в клиент для импорта серверов.
Для v2rayNG: нажмите + > "Импорт из буфера обмена".
Для NekoRay: выберите Server > Add profile from clipboard.
Для NekoRay Android: нажмите ⋮ > в правом верхнем углу кнопка добавить > тип группы "Подписка" > сортировка "По задержке" > ссылка для подписки (см. Доступные виртуальные сервера). После чего создать группу через кнопку справа вверху и дождаться добавления конфигов.
Для Hiddify: нажмите + > "Добавить из буфера обмена".
3. Отсортируйте полученные сервера по задержке с помощью функционала клиента.
Для v2rayNG: выберите "Время отклика профилей группы" (подождите несколько минут, это зависит от мощности устройства и количества серверов) > "Сортировка по результатам теста".
Для NekoRay: выберите Server > Current Group > Url Test (ждите). Затем отсортируйте по вкладке Test Results.
Для NekoRay Android: нажмите ⋮ > URL Тест (дождитесь окончания) > Сортировка > По задержке.
Для Hiddify: сортировка происходит в реальном времени, что может нагрузить систему.
4. Проверьте функциональность сервера через веб-ресурсы: https://browserleaks.com/dns, https://2ip.io/privacy.

Серьезно и безотлагательно отнеситесь к проверке конфигурации, которую планируете использовать. Обследуйте свою виртуальную сеть и проанализируйте SSL-трафик через ресурс https://www.doileak.com/classic.html. Если вы заметили проблемы с SSL, рекомендуется сменить сервер.
Поскольку протокол VLESS не шифруется напрямую, трафик может передаваться через TLS. Наличие TLS является показателем безопасности. Если конфигурация не имеет показателя TLS, стоит усомниться в защищенности соединения. Следовательно, если вам необходимо безопасное соединение, проверяйте конфигурации серверов на наличие TLS; такие серверы более безопасны.
Рекомендуется периодически обновлять список серверов (группу профилей или подписку группы и т. п.), так как сервера могут истекать или становиться неактуальными со временем.
Если сервер активен, это еще не означает, что он функционирует в текущих условиях сети, где система цензуры постоянно анализирует трафик. Проверяйте IP через браузер.
TLS - протокол, используемый поверх протокола VLESS для улучшения безопасности и конфиденциальности соединений.
SSL-трафик в контексте VPN относится к использованию протокола SSL (Secure Sockets Layer) для шифрования данных, передаваемых через виртуальную частную сеть. Это обеспечивает безопасность и конфиденциальность данных, передаваемых между клиентом и сервером.
